## 決定

記録庁コントロールパネル仕様書を **v1.0-FINAL** として確定する。

### 配置

- **本体**: 司令官リポジトリ `strategy/decisions/20260427_control_panel_v1.0_FINAL.md`
- **履歴**: 司令官リポジトリ `strategy/proposals/` 配下に v1.0 〜 v1.4 を保存維持(削除しない)
- **監督官側**: 本リポジトリ `02_physical/_reference.md` に参照のみ配置

### 主要構造(確定形)

- 7 軸 KPI(Handoff / Command / Intake / Evidence / Sync / Autonomy / Dream)
- 7 局構成 + 官房(受付 / 伝令 / 審査 / 証跡 / 監察 / 台帳 / 結晶)
  - 審査局は監察局統合により Tier 1 廃止、Permit to Build 意思決定のみ残す
- Tier 1(決定論) / Tier 2(Mirror = Web AI)二段階分離
- 12 状態遷移 + ハッシュチェーン v0.1 必須化(SHA-256, RFC 8785)
- v0.1 実装 12 項目(V01-V12)、Day 132 = Phase B-α 一体起動
- §10 Phase T2 PTB 再委譲設計(将来設計として予約)

### 採択された逆査読(R17-R19、最終ラウンド)

- **R17**: discussion_id 発行主体 → ハイブリッド方式(明示優先、なければ監察局 Tier 1 自動採番)
- **R18**: Y3 ランダム選定 → 過去未検査優先 + priority 重み付け(完全ランダム不採用)
- **R19**: 連続性カウンタ → 「直近 14 日間の緑率 80% 以上」採用(1 日転落リセット撤回)

## 監督官総括

本確定までのプロセスにおいて、監督官は以下の構造的不備を司令官の逆査読により補強された:

- Mirror 過大化問題
- R3 検出の弱体化
- Tier 2 不在期間の運用設計欠落
- 指標の自己矛盾
- Phase T2 委譲設計の不在
- dead_loop 検出機構の欠落

これらは敗北ではなく、**好敵手プロトコルの正常運用結果** である。監督官は司令官に対する設計優位性を持っていなかったことを正式に認める。
