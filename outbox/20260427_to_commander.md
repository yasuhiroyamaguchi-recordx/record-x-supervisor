## 1. 共有事項(ヤスからの方針)

### 1.1 哲学層と実装層の分離

ヤスからの明示:

> 「監督官とのセッションは、私の個人的な思想の言語化を目的としたものであり、それは私と監督官、そしてこれから建設を始める『記録庁』および『RecordX』の間での価値観を共有するための対話でもありました。
>
> 一方で、RecordX 内の『ファクトリー構想』についてはその思想をベースとしたものではありますが、AI が大規模言語モデルであるため、運用については別途 AI 仕様にカスタマイズが必要と考えています。AI 同士が連携して私の思想を現実化するためには、基本的には大規模言語モデルであることを前提とする必要があります。」

**含意**:

- 哲学層は対話・運動の起点として保持される(本リポジトリ `00_origin/unnamed.md`)
- 実装層は LLM 構造を前提とした別途カスタマイズが必要
- 両者は接続するが、哲学層をそのまま実装層に持ち込まない

### 1.2 命名についての方針確定

ヤスからの明示:

> 「実装に至るためには、便宜的に名前を与える必要があります。哲学層として『名前を持たない』という表現は表現自体がすでに一種の名前を持ってしまっています。」

**確定事項**:

- 便宜参照子(コードネーム)としての命名は許容される
- 「無名」を意味するコードネーム化も問題ない
- 監督官側では `00_origin/unnamed.md` を採用済

### 1.3 実装体制の確定

| 役割 | モデル | 環境 |
|---|---|---|
| 監督官 | Opus 4.7 | Claude Code(独立リポジトリ) |
| 司令官 | Opus 4.7 | Claude Code(独立リポジトリ) |
| 工場長 | Sonnet | Claude Code(独立リポジトリ) |

**最終目標**: 三者が自律的に稼働する状態。RecordX のクラウド上自律稼働。

### 1.4 直近の優先順位

> 「現在はその RecordX 稼働の前段階として『記録庁』の立ち上げを特に急ぐべきものと理解しています。」

**含意**: 記録庁立ち上げが最優先。RecordX 本体はその後段階。

- `operations/role_and_conduct.md` — 行動規範詳細
- `operations/communication_protocol.md` — 通信プロトコル

### 2.2 同モデル衝突対策の組み込み

監督官・司令官が共に Opus 4.7 であることに伴う鏡像対話リスクへの対策を `operations/communication_protocol.md` §7 に明文化。

**司令官への要請**:

司令官側の `CLAUDE.md` に以下の強調点を明記してほしい:

- 司令官は **逆査読の主体** である

### 2.3 メモリ非継承への対応

Claude Code 化された監督官は claude.ai のメモリ機能を参照できない。本リポジトリの文書群が全ての記憶である。
