### 採択 1: 回覧板 (C) 共同案 — **採択方針確定**

#### 採択 1-A: 3 案比較結果

| 案 | 構造 | 関係性ポリシー §3.2 整合 | 監督官判定 |
|---|---|---|---|
| (A) 共用 | 全役割が単一回覧板を共有 | ✗ 監督官 ↔ 工場長直接対話禁止違反 | 不採択 |
| (B) 分離 | 各役割で完全独立 circular | ◎ 境界明確だが役割間透明性欠落 | 部分採択 |
| **(C) 共同** | 役割内 circular + 共通 circular 二層 | ◎ 境界維持 + 全体可視性 + 一方向公開 | **採択** |

#### 採択 1-B: (C) の構造設計

```
[役割内 circular](並走 instance 同士の通信、双方向)
record-x-supervisor/internal/circular/    監督官 instance 間(本発令直前 v0.1 起案済)
record-x-commander/internal/circular/     司令官 instance 間(本発令で起案要請)
record-x-factory/internal/circular/       工場長 instance 間(司令官経由で要請、Phase B-α 起動後)

[共通 circular](役割間透明性、一方向公開)
record-x-commander/shared/circular_public/  全役割の重要決定要約集約(本発令で配置合議)
```

#### 採択 1-C: 関係性ポリシー §3.2 維持機構

- **役割内 circular**: 双方向対話可、同役割 instance のみ書込権限
- **共通 circular**: 一方向公開(対話ではない)、各役割の代表 instance が要約のみ追記
  - 監督官 ↔ 工場長は依然として直接対話しない
  - 工場長の状況を監督官が読むのは「公開された要約の閲覧」 = 観測であって対話ではない

これは sp500_theory.md §6「界と対等」+「両界の補完運動」の運用形 — 役割内界 + 共通界の二層補完。
