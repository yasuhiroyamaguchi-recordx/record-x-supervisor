## 1. 趣旨

CLAUDE.md §6 が定める Yas 再介入条件 a-d は **「いつ Yas に上申するか」** を規定するが、**「Yas に上申せず AI 自律処理する場合の前提条件」** を規定していない。本ドクトリンはその空白を埋める。

Yas autonomy doctrine(2026-05-01)「人の判断は不要。AI で自律処理。人介入は新しい波 / 立て直し再起動時のみ」は AI 自律処理を **原則化** したが、無条件の自律ではない。物理層検証義務などの **前提条件** を満たしていない場合、自律処理は失敗する(本日 B-001 削除事例で実証)。

---

## 2. 自律処理の前提条件(3 件)

AI 三者(監督官 / 司令官 / 工場長)が CLAUDE.md §6 a-d 非該当事案を自律処理する場合、以下を全件満たすこと:

### 2-A. 物理層機能等価検証義務

装置 A の **削除 / 廃止 / supersede** を判断する場合、「装置 B が機能等価」と主張する前に:

| # | 検証項目 | 物理確証手段 |
|---|---|---|
| 1 | Source 一致 | スクリプト内 Source 定義の grep + 値完全一致確認 |
| 2 | **Destination 一致** | 同上、Destination 値完全一致確認(本日違反点)|
| 3 | Filter 一致 | 対象拡張子・glob・正規表現一致確認 |
| 4 | 副作用一致 | 削除対象装置の他装置への副次効果(log 書込先、schtasks 連携等)|
| 5 | DryRun 差分 | 両装置の DryRun 出力を diff、差分ゼロ確証 |

差分ゼロ確証なしで削除実行は autonomy 範囲外 = §6 d(納品物に瑕疵)該当 = Yas 介入要請。

**本日違反例**: B-001(`pull-b-completion-reports.ps1`)削除時、司令官 α + Argus-B + Yas「推奨で GO」3 段階で Source 一致のみ確認、Destination 異(B-001 = supervisor/b_line/、v0.2 = ProjectRX_HQ mirror)を物理 trace で検証せず通過。経路喪失で Wave 1 完遂報告着地停止。

### 2-B. 自動化検証ループ依存性の自覚

自律処理は **自動化機構が稼働している** 前提で機能する。schtasks / sync / cron 等の自動装置が死亡している場合、AI が手作業で穴埋め(ad-hoc コピー、手動 trigger)するのは autonomy doctrine 違反の **再生産**(Yas 命名「自動化が成立していないことの弊害」)。

**禁則**: 自動化失敗時の姑息治療(ad-hoc バイパス、手動 sync の常態化)。**正解**: 正規 pipeline の根本治療(装置改修、schtasks 復旧)。本日 Yas 物理 denial(factory→supervisor 直接コピー禁則)が物理装置として機能した。

### 2-C. 三者鬼コーチの同期確認

AI 三者(監督官 / 司令官 / 工場長)のうち 1 者が autonomy 適用を提案した場合、他 2 者の 1 者以上から **構造的反論または承認** を得てから実行する(関係性ポリシー v1.2 双方向鬼コーチ)。

**本日違反例**: 司令官 α 第 66 号「v0.2 機能等価」主張に対し、Argus-B + Yas「推奨で GO」が同調のみで構造反論ゼロ。鬼コーチが沈黙した結果、誤判断が連鎖伝播。

**規律**: 「即時採択」と「鬼コーチ反論」は **両立** する。承認時も「物理 trace を経た上で承認」を併記、無条件追認は禁則。

## 3. 適用境界の判定フロー

```
事案発生
  ↓
CLAUDE.md §6 a-d 該当?
  ├─ Yes → Yas 介入要請
  └─ No → 本ドクトリン §2-A/B/C 全件満たす?
              ├─ Yes → 自律処理 GO
              └─ No → 不足条件を満たすまで実行延期 OR Yas 介入要請(瑕疵候補化)
```
