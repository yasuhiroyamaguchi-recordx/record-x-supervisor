### 1-C. 層間の関係性

```
[フロント層]  即時対応 + 議論生成
     ↓                              ← (上昇:即時判断 → 過去判断 + 知識化)
     ↓ ドリーム機能(継続抽出パイプライン)
     ↓                              → (下降:蓄積知識 → 即時判断の参照素材)
[バック層]   過去判断 + 蓄積知識
```

**双方向関係**:
- **上昇**: フロント層の即時判断 → ADR / EVT / B-番号 / CP-番号として正式化、バック層に蓄積
- **下降**: バック層の哲学層 + 規範層 + 過去判断 → CLAUDE.md 起動時 + 各セッションでフロント層の判断素材

これは **NTT 記事 §3 の「フロント / バック分離」+ §6 「2 週間運用効果」(レビュー速度 1 時間 → 20 分)の物理層実装**。

= **NTT 記事の 2 層ナレッジ基盤バック層継続抽出を三者展開** で物理装置化(ADR-009 §6 三者統合自動化)。

### 2-B. 結晶化の方向性(層間の物理経路)

```
[フロント層]
  ├ 監督官 A 即時対応 → 第 N 次発令(outbox/)+ EVT 記録(error_patterns.md)
  ├ 円卓会議 → council 議事録 + B-番号台帳
  └ sync-tickets → tickets_completed/
                                              ↓ ドリーム結晶化(日次自動)
                                              ↓ 直近 24h の更新を集約
                                              ↓ ADR 候補 + EVT 候補 + 哲学層拡張候補に LLM 結晶化
                                              ↓ ヤスレビュー(検診プロトコル)
[バック層]
  ├ ADR(正式採択)
  ├ 哲学層エントリ(運動の起点として追記)
  └ EVT 系列(構造的盲点として追記)
```

### 2-C. NTT 記事との対応(物理装置 vs 概念)

| NTT 記事 §3 | 記録庁 |

| 週次/隔週の継続的抽出 | ドリーム結晶化日次自動(三者統合スケジュール、ADR-009 §6)|
| decision.md | B-番号台帳 + P-番号台帳 + ADR + EVT-番号台帳 + 過去発令履歴(本ドキュメント §1-B-1)|
| knowledge.md | 哲学層 + 関係層 + 物理層参照 + 規範層 + rubrics + CLAUDE.md(本ドキュメント §1-B-2)|
| 人間レビュー | ヤス検証(`role_and_conduct.md` §1.1-E 検証必要度マーキング義務適用) |
