## 2. 指示 2-A: 構造的反論(軽微注記 1 件)

### 2-A-1. 軽微注記 P6: Layer 0 自動レビューの「観測 + 軽微フィードバック」境界の明文化要請

#### 注記内容

第 14 次発令 1-C「Layer 0 自動レビューは観測 + 軽微フィードバックに留め、構造方針判断は監督官手動セッションで実施」は **方針として正しい** が、「構造方針」と「軽微フィードバック」の **境界判定基準が未定義**。

具体例:
- ✅ 軽微フィードバック想定: チケット内 lint 違反、命名規則違反、テスト件数不足 → verdict APPROVE_WITH_NOTES 自動発出
- ⚠ 境界曖昧: forbidden_paths への近接(直接違反ではないが構造的懸念)→ APPROVE_WITH_NOTES か REQUEST_CHANGES か
- ❌ 構造方針判断想定: ADR レベルの設計逸脱、関係性ポリシー違反 → 必ず手動セッションへエスカレーション

Layer 0 自動レビューが「⚠ 境界曖昧」を判定する際の具体ルールが未確立。

#### P6 提案: 「自動 → 手動エスカレーション基準」を `operations/communication_protocol.md` §3.2 に追記要請

監督官側の `operations/communication_protocol.md` §3.2 に以下を追記する形での提案:
- Layer 0 自動レビューが REQUEST_CHANGES / REJECT を発出する場合 → 必ず手動セッションへエスカレーション(自動発出禁止)
- APPROVE / APPROVE_WITH_NOTES のみ Layer 0 自動発出を許容
- 境界判定で迷った場合 → デフォルトで手動セッションへエスカレーション(安全側)

これにより「構造方針判断は手動セッション」原則が物理装置で担保される。
