# Circular 20260429_006: 構造観察 — Layer 2 schtasks 22 分間隔誤登録 + 二重登録(EVT 候補)

## 起案趣旨

ヤスから提示された Layer 2 自律巡回ログ(2026-04-29 13:47:54 + 14:10:02)から、**Layer 2 タスクスケジューラが設計値(6 時間)から大きく乖離した 22 分間隔で起動している** ことが判明。加えて `\Record-X-Layer2-Strategy` + `\RX-Layer2-Strategy` の **二重登録** も発見。

ヤスの作業集中力への悪影響 + Phase B-α 完全自動化(commit `8ec775a`)の運用衛生不全を構造観察として記録 + A2(マスター = `archive/error_patterns.md`)+ 司令官 α(該当 schtasks の起案責任主体)に通知する。

A1 直接編集は二重起案再発リスクのため、L1 circular 経由で通知する形 = circular_20260429_002 / 004 / 005 と同方針。

## §1. 観察事実(v1.0 → v1.1 改訂、Yasu GUI スクショで真因確定)

### 1-A. ログ間隔(ヤス端末で観察、初期)

| ログ時刻 | 間隔 |
|---|---|
| 2026-04-29 13:47:54 | - |
| 2026-04-29 14:10:02 | **22 分 08 秒** |

設計値(DO-COMMANDER-007 v2 確定):
- Layer 1(実装層、HQ): 10 分
- **Layer 2(戦略層、Commander): 6 時間**(00:00/06:00/12:00/18:00 JST)

ログ表記「Layer 2 自律巡回 v1 サイクル開始」と起動間隔 22 分は **不一致**。

### 1-B. 二重登録(初期 schtasks /query 結果)

| タスク名 | 次回実行 | 状態(無効化前)|
|---|---|---|
| `\Record-X-Layer2-Strategy` | 2026-04-29 19:47:52 | 準備完了 |
| `\RX-Layer2-Strategy` | 2026-04-29 18:50:00 | 準備完了 |

両タスクとも 6 時間サイクル(`<Repetition><Interval>PT6H`)= **真因ではない**。

### 1-C. 真因確定(Yasu GUI スクショ、6 タスク全件)

| タスク | 状態(本日 disable 後)| トリガー | 次回 / 前回 |
|---|---|---|---|
| `\Record-X-Layer2-Strategy` | 無効 | 04-29 01:47 + **6 時間ごと** | 19:47:52 / 13:47:53 正常終了 |
| `\RX-Layer2-Strategy` | 無効 | 04-29 00:50 + **6 時間ごと** | 18:50:00 / 12:50:01 正常終了 |
| **`\RX-Layer1-Implementation`** | **無効**(本日 disable)| 04-29 00:50 + **10 分ごとに無期限** | **15:20:00 / 15:10:01 現在タスク実行中(0x41301)** |
| `\RX-Layer3-News` | 無効 | 毎日 0:50 起動(12 時間ごと?)| 04-30 0:50 / 1999-11-30 未実行 |
| `\RX-Layer4-Checkup` | 無効 | 04-29 00:50 + **12 時間ごと** | 04-30 0:50 / 12:50:01 正常終了 |
| `\RecordX_NightlyFlight` | 無効 | **毎日 02:10 起動**(未認知タスク)| 04-30 02:10 / 02:10 正常終了 |

### 1-D. 真因確定:`\RX-Layer1-Implementation` が 10 分間隔の元凶

**`\RX-Layer1-Implementation` = 2026-04-29 00:50 起動 + 10 分ごとに無期限繰返**。

これが 22 分間隔起動の真犯人(2 回分の 20 分 + 2 分誤差で観察)。

ログ表記「**Layer 2** 自律巡回 v1 サイクル開始」と稼働実体「**Layer 1** タスク = RX-Layer1-Implementation」の **命名と動作の完全な不一致** が確定。

考えられる仮説:
- Layer 1 タスクが Action として `layer2_entry_point.ps1` を呼び出している(設計バグ or 命名ミス、要確認)
- または Layer 1 / Layer 2 が同じ entry_point スクリプトを使い回し + ログ表記が固定文言

### 1-E. 追加観察:RecordX_NightlyFlight(未認知タスク)

`\RecordX_NightlyFlight` = 毎日 02:10 起動 = **A1 事前査読外の未認知タスク**。命名規則(`RecordX_` アンダースコア)が他タスク(`Record-X-` ハイフン / `RX-` ハイフン)とも異なる = **別起案者・別時期の登録疑い**。本タスクの内容(Action コマンド)+ 起案者は要確認。

## §2. 仮説(初期 → 確定、Yasu GUI スクショで真因特定)

### 2-A. 初期仮説(却下含む)

| 仮説 | 重要度 | 結論 |
|---|---|---|
| (I) Layer 2 Repetition Interval 誤値 | - | **却下**(両 Layer 2 共 PT6H 確認)|
| (II) Layer 2 二重登録の合成 | - | **却下**(2 タスク同 6H 周期、22 分起動不可)|
| (III) Layer 2 StartWhenAvailable + Repetition 二重定義 | - | **却下** |
| (IV) layer2_entry_point.ps1 内部ループ | - | **却下**(281 行精査、ループ無し)|

### 2-B. 確定真因

**`\RX-Layer1-Implementation`(10 分間隔タスク)がログ「Layer 2 自律巡回 v1」と表記される処理を起動していた**。

| 確定要素 | 根拠 |
|---|---|
| Layer 1 タスクが 10 分間隔で稼働 | Yasu GUI スクショ「2026/04/29 00:50 に起動 - トリガーされた後、10 分間ごとに無期限に繰り返します」|
| 22 分間隔ログは Layer 1 起動の連続 | 13:50 + 14:00 + 14:10 のうち 13:47:54 と 14:10:02 をログで捕捉(2 分誤差含む)|
| 「Layer 2」表記の不一致 | ログ固定文言 or Layer 1 の Action が `layer2_entry_point.ps1` を呼出 |

### 2-C. 残課題(構造的調査要)

- **(VII) Layer 1 タスクの Action コマンド確認**(Layer 1 が Layer 2 entry_point.ps1 を呼出している場合、設計バグ確定)
  - `schtasks /query /tn "\RX-Layer1-Implementation" /xml` で `<Exec><Command>` + `<Arguments>` 確認要
- **(VIII) ログ表記の固定文言問題**(Layer 1 起動なのに「Layer 2 自律巡回 v1」と固定で書く設計)
  - layer2_entry_point.ps1 v2 内 `Write-Log "=== Layer 2 自律巡回 v1 サイクル開始 ==="` の文言は Layer 2 専用 = Layer 1 から呼ばれるなら誤った文言
- **(IX) RecordX_NightlyFlight の起案者・内容確認**(A1 事前査読外タスク)
