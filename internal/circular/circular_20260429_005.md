## 起案趣旨

監督官 A1 は本日 Day 131 朝、ヤス指示「閾値判定で、馴れ合いにならないようなフィードバックが必要」(2026-04-29 朝)を受けて、`operations/post_build_gate_checklist.md` を **v0.1 → v0.2 改訂**:

- §1 12 項目を **重要度別重み付け P0-P3 + 部分点(0/0.5/1)** に改訂
- §6 新設「個別フィードバック発令前の品質ゲート閾値判定」= 6 軸閾値(A 構造的反論件数 / B 改善事案提案件数 / C APPROVE 連続発出 / D 構造的反論なし発令率 / E DA ラウンド実施率 / F 形式採択化検知)

A1 直接編集は二重起案再発リスクのため回避し、L1 経由で要請する形 = circular_20260429_002 / 004 と同方針。

## §1. 補強要請内容

### 1-A. 既存 auto-evt-recorder v0.7 のカバー範囲

A2 起案 auto-evt-recorder.ps1 は R1-R7 の drift 検出ルールを実装済(SITREP_20260429.md v1.1 ステップ 31-37 + 39 参照):

- R1 approve_streak(月次集計、verdict APPROVE 連発)
- R2 role_total_red_streak
- R3 order_pace_overrun
- R4 sync_log_warnings
- R5 escalation_file_emerged
- R6 frontmatter aware
- R7 P-number registry

これらは **月次 / 日次 / セッション単位** の集計検知。

### 1-B. 不足:**個別フィードバック単位** の馴れ合い兆候検知

post_build_gate_checklist.md v0.2 §6 で定義した 6 軸閾値のうち、**E. DA ラウンド実施率** + **F. 形式採択化検知** は個別発令単位で検知する必要がある:

- DA ラウンドの反論件数が機械的に毎回 3 件ジャスト = ロールプレイ化兆候
- 反論内容が「文面修正」「typo」レベル = 構造度低下
- 即時採択候補化判断が「全件 retain_with_grounding」連続 = 演技反論

これらは個別発令の **frontmatter / DA round metadata** を直接検査する必要があり、月次集計では検知できない。

### 1-C. R8 ルール追加提案

A2 マスター `auto-evt-recorder.ps1` v0.8 改訂で以下を追加:

```yaml
R8_per_feedback_drift:
  description: 個別フィードバック発令単位での馴れ合い兆候検知(post_build_gate_checklist.md v0.2 §6 連動)
  applicable: per outbox/*.md 発令(scale: medium 以上 + verdict in [APPROVE, REQUEST_CHANGES, REJECT])
  data_source:
    - outbox/*.md frontmatter(verdict / scores / findings)
    - staging/self_check_log.jsonl の devil_advocate エントリ(対応する outbox 発令と紐付け)
  signals:
    R8_a_no_findings_when_request_changes:
      condition: verdict in [REQUEST_CHANGES, REJECT] AND findings.count == 0
      severity: yellow
      evt_category: audit_overblock
    R8_b_findings_only_p3:
      condition: verdict == APPROVE_WITH_NOTES AND all findings.severity == "P3"
      severity: info
      evt_category: drift_warning(軽微指摘のみ = 形式採択化兆候)
    R8_c_da_count_uniform:
      condition: 直近 3 件発令(scale >= medium)で DA round.weakness_count が全て 3 件ジャスト
      severity: yellow
      evt_category: formal_acceptance_detection(機械化兆候)
    R8_d_da_wording_only:
      condition: 直近 3 件発令で DA round.weaknesses[*].structural_grounding == "filling_only" 連続
      severity: red
      evt_category: formal_acceptance_detection(本気度欠如)
    R8_e_da_all_retain:
      condition: 直近 5 件発令で DA round.weaknesses[*].immediate_adoption_decision == "retain_with_grounding"(withdraw / revise なし)
      severity: yellow
      evt_category: drift_warning(演技反論)
    R8_f_no_da_when_required:
      condition: verdict in [APPROVE, REQUEST_CHANGES, REJECT] AND scale in [medium, large] AND DA round 不在
      severity: red
      evt_category: audit_underblock(DA 装置の機能不全)
  output:
    target: archive/error_patterns.md (auto-EVT 起案)
    target_jsonl: sync/checkup-scores/evt_telemetry/{date}.jsonl
    notification: red 判定時、staging/.three_party_halt.lock 候補 + Yasu 通知(notify-yasu-email)
```

### 1-D. post_build_gate_checklist.md v0.2 §6-A の閾値との整合

| post_build_gate v0.2 §6-A 軸 | R8 サブルール |
|---|---|
| A 構造的反論件数 | R8_a |
| B 改善事案提案件数 | (DA round metadata 連動)|
| C APPROVE 連続発出 | 既存 R1 で月次集計、R8 は追加しない |
| D 構造的反論なし発令率 | R8_b(個別単位)|
| E DA ラウンド実施率 | R8_f |
| F 形式採択化検知 | R8_c / R8_d / R8_e |

R8 は post_build_gate_checklist.md v0.2 §6-A の **個別発令単位への落とし込み**。両者は補完関係。
