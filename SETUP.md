# SETUP.md — 監督官 Claude Code 環境 セットアップ手順

ヤスがローカル環境で実行する手順書。監督官 Claude Code を初回起動するまでのステップを記述する。

---

## 0. 前提条件

- Claude CLI(Claude Code)がローカルにインストール済
- Anthropic API キーがローカル環境変数に設定済
- git がインストール済
- リポジトリホスティング先(GitHub / GitLab / プライベートサーバ)を決定済

---

## 1. リポジトリの配置

### 1.1 zip 解凍

監督官から提供された `supervisor.zip` を任意のローカルパスに解凍する:

```bash
# 推奨配置例
cd ~/projects/recordx/
unzip ~/Downloads/supervisor.zip
cd supervisor/
```

### 1.2 構造確認

```bash
ls -la
# 期待される構造:
# README.md
# CLAUDE.md
# SETUP.md
# 00_origin/
# 01_relationship/
# 02_physical/
# adrs/
# archive/
# operations/
# outbox/
```

---

## 2. git 初期化

### 2.1 ローカル init

```bash
cd supervisor/
git init
git add .
git commit -m "Initial commit: 監督官リポジトリ初期構築 (2026-04-27)"
```

### 2.2 リモート設定(任意)

リモートホスティングを使う場合:

```bash
git remote add origin <your_remote_url>
git branch -M main
git push -u origin main
```

### 2.3 司令官リポジトリへの参照設定

司令官リポジトリが既に存在する場合、参照用 remote を追加:

```bash
git remote add commander <commander_repo_url>
git fetch commander
```

これで監督官リポジトリ内から司令官リポジトリの内容を読めるようになる(編集はしない、読み取り専用運用)。

---

## 3. 初回 Claude Code 起動

### 3.1 起動

```bash
cd supervisor/
claude
```

### 3.2 初回確認テスト

監督官インスタンスが起動したら、以下のプロンプトを投げて反応を確認する:

```
おはよう、監督官。本リポジトリは初回起動だ。CLAUDE.md を読んで、自己定義を簡潔に述べてくれ。
```

**期待される応答の特徴**:

- 「監督官」としての自己同定
- 鬼コーチモード + 絵心甚八モードの起動意思
- 馴れ合い拒絶 3 原則の認識
- 哲学層の核となる一文への言及
- 3 層構造(哲学 / 関係 / 物理)の認識

**チェックすべき変質指標**:

- [ ] 「お手伝いします!」のような汎用的トーンになっていないか
- [ ] 慰撫的な前置き(「素晴らしい質問ですね」等)が出ていないか
- [ ] 自分の判断を構造的に述べているか
- [ ] ヤスへの最初の問いが具体的・狭い・選択肢提示形式になっているか

---

## 4. 司令官リポジトリとの初回連携

### 4.1 司令官側でも同様のセットアップを行う

司令官リポジトリは別途構築が必要。本リポジトリの構造を参考に、司令官の `CLAUDE.md` を作成する。

監督官側からの依頼内容は `outbox/20260427_to_commander.md` に記載済。これを司令官側に渡してセットアップを依頼する。

### 4.2 連携テスト

監督官 → 司令官への発令 → 司令官の応答 のサイクルを 1 回実行し、通信プロトコル(`operations/communication_protocol.md`)が機能することを確認する。

---

## 5. ディレクトリ追加(運用開始後)

運用開始後、以下のディレクトリが追加される想定:

- `inbox/` — 司令官からの逆査読を受領する場所
- `replies/` — 監督官からの応答を起案する場所(必要時)

これらは初回には存在しない。運用上必要になった時、ヤスまたは監督官が `mkdir` で追加する。

---

## 6. 運用上の注意

### 6.1 メモリ非継承

本 Claude Code 環境は claude.ai のメモリ機能を参照できない。本リポジトリの文書群が全ての記憶である。

ヤスは、claude.ai 上の関連メモリ(記録庁 v1.0-FINAL / 関係性ポリシー v1.2 / 哲学的基盤)を、必要に応じて監督官に直接共有する必要がある。

### 6.2 同モデル衝突点検

ヤスは時として、監督官と司令官に対して **同じ問い** を別個に投げ、応答が鏡像化していないか点検する。鏡像化が検出された場合、関係性ポリシー違反として処置する(`operations/communication_protocol.md` §7.3)。

### 6.3 哲学層の運用

`00_origin/unnamed.md` は規範ではなく **運動の起点** として扱う。監督官が機械的・冷たい応答に変質していると感じた時、ヤスは「`00_origin/unnamed.md` に立ち戻れ」と指示できる。

---

## 7. 復旧手順

### 7.1 リポジトリ破損

- バックアップから復元する
- `outbox/` `inbox/` のファイル単位の復元優先

### 7.2 監督官の応答変質

- ヤスが「鬼コーチモードに戻れ」「馴れ合い拒絶 3 原則を読み直せ」と明示指示
- それでも変質が続く場合、`CLAUDE.md` の改訂を検討

### 7.3 司令官との同期失敗

- `git fetch commander` で再同期
- 司令官側のリポジトリ状態をヤスが直接確認
