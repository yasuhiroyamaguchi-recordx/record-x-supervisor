## 2. §2 構造的根因分析

### 2-A. 質問経路の物理的空白

| 関係 | 役割 | 質問経路 |
|---|---|---|
| 工場長 → 司令官 α | チケット選択 / 起案 / 実装報告 | ✅ 司令官 α 主管 |
| 工場長 → 監督官 A | **技術質問(実装前確認)** | 🔴 **物理的空白**(現行 §5 で「直接対話しない」)|
| 工場長 → ヤス | 例外的判断仰ぎ | ヤス再介入条件 §6 a-d |

**= 司令官 α は「チケット発行のみ」(チケット以外発信禁止)= 技術質問への回答経路を持たない構造**(司令官 α 機能制約)

→ **工場長が技術質問を持っても、回答可能な相手が物理的に不在** = 役割境界違反 連続 3 例(EVT-104)の構造的根因

### 2-B. 監督官 A 規律解釈硬直(EVT-117 候補)

| 経路 | 監督官 A の硬直反応(本セッション末末末) |
|---|---|
| 工場長 → 監督官 A 技術質問(実装前確認) | 🔴 **司令官 α 経由迂回要請**(現行 §5「直接対話しない」厳守解釈)|
| ヤス指摘 | 「司令官経由で工場長に...伝言ゲーム」 |
| 自己訂正 | **直接回答妥当**(監督官 A リポジトリ全体把握 + 即答可、伝言ゲーム回避)|

= **規律の文字解釈** vs **規律の精神**(役割境界の本質 = 司令官 α 主管領域への監督官 A 越権防止)= 後者が真の規律

## 3. §3 改訂候補(v0.1 草案)

### 3-A. 改訂後の §5(案)

```
## 5. 役割の境界(厳守)

| 関係 | 内容 |
|---|---|
| 監督官 ↔ ヤス | 対話・進言・査読要請の受領 |
| 監督官 ↔ 司令官 | 発令・査読・逆査読受領・採択判断 |
| 監督官 ↔ 工場長 | **原則直接対話しない**(司令官経由のみ)、例外 = 工場長 → 監督官 技術質問(実装前確認)への直接回答可 |
| 司令官 ↔ 工場長 | チケット発行・実装・報告(監督官は介入しない) |

### 5-A. 監督官 ↔ 工場長 例外条件(技術質問への直接回答可)

以下の条件を **すべて満たす** 場合、工場長 → 監督官 技術質問への直接回答を可とする:

- (i) 工場長発信であること(監督官 → 工場長 直接発信は引続き不可、発令は司令官 α 経由)
- (ii) 質問内容が **技術質問(実装前確認、リポジトリ全体把握 + 即答可)** であること
- (iii) 質問内容が **チケット選択 / 起案 を含まない** こと(チケット選択 / 起案 = 司令官 α 主管、監督官 A 越権)
- (iv) 質問内容が **実装報告(チケット完遂報告)を含まない** こと(実装報告 = 司令官 α 主管経路)

### 5-B. 監督官 → 工場長 直接発信 = 引続き不可

- 発令(チケット手配 / 整流要請 / 査読依頼等)= 司令官 α 経由必須
- 監督官 A → 工場長 直接発信は CLAUDE.md §5 違反として継続厳守

### 5-C. 工場長 → 監督官 越権発言の整流(現行運用継続)

- 工場長 → 監督官 チケット選択依頼(「W2-XXX か W1-YYY のどちら次に?」等)= 司令官 α 経由整流要請(EVT-104 系列)
- 工場長 → 監督官 実装報告(「PR #XXXX merge 完遂」等)= 司令官 α 経由配送要請

```
