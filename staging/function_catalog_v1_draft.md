## §4. 補正運用(過剰加速リスク対処、Mamaz 補正ポイント同型)

### 4-A. 速度補正(本日連鎖 11 例 + EVT-083 起源)

Plan-First 物理事例第 1-11 例本日連鎖 = 過剰加速リスク = 検証期間 1 日目で EVT-083 発生 = **「速度 ≠ 規律」物理証拠**

→ Mamaz 報奨金「13.00 以上の減算ならないように」補正運用同型適用:

| 達成パターン | 補正運用 |
|---|---|
| 連続 2 日 物理事例 +5 件超 | 3 日目で意図的減速 + 検証期間内振り返り強制 |
| 累積物理事例 +25 件超(過去同期間)| 補正ポイント = 累積件数 - 25 を次期間スコアから減算 |
| EVT 候補連鎖 1 日 5 件超 | 翌日強制振り返り + Plan-First 厳格適用 |

### 4-B. 補正の意図

「規律装置完成 ≠ 規律機能」の物理証拠継続 = 速度 = 規律と誤認しないため。Mamaz 補正運用は「持続性確保のため意図的減速」= AI でも同型適用。

## §5. Q&A(想定質問)

### Q1. 全体で healthy 達成しないと個別評価対象外?

**A**: いいえ。全体 concerning でも個別 healthy 達成領域は個別評価可(Mamaz 同型「全体 11.00 不達でも単独 11.00 達成は対象」)。

### Q2. 検証期間中に EVT 起案された場合は対象外?

**A**: いいえ。EVT 起案 = 自己訂正サイクル健全性の物理証拠 = 評価対象内、ただし暫定スコア化(自己訂正完遂後に確定スコア)。

### Q3. ヤス指示「即実装」発動時の Plan-First 例外は減点?

**A**: いいえ。例外条件 (i)-(iv) 該当発動 = Plan-First 規律遵守(透明性確保で完遂報告明示)= 評価対象外、ただし例外発動頻度 50% 超は別系列で警告。

### Q4. 工場長 Castor の DO-COMMANDER-030 blocked = 減点?

**A**: いいえ。前提疑い経路 = SS_GUARD §DO-541 直接該当 = Plan-First 規律遵守の物理事例 = 評価加点。blocked 報告自体が誠実申告 = Case-7 遵守。
